空状态卡片的设计原则和各卡片类型的差异处理：

设计原则
原则说明
保留占位不自动删除卡片，保留在笔记原位置，让用户知道「这里曾经有个东西」
降低视觉权重灰色调 + 虚线边框 + 浅背景，和正常卡片形成明显区分，不抢注意力
保留原始信息副标题保留原文件名/原时长/原域名，方便用户回忆是什么内容
可操作长按可替换或移除，不是死的占位
各卡片类型差异
卡片类型空状态文案副标题保留icon
文档卡片文件已被删除原文件名 + 格式灰色文档 icon
音频卡片录音已被删除原录音名 + 时长灰色麦克风 icon
网页卡片网页无法访问原域名 + 路径灰色地球 icon
未知类型当前版本暂不支持查看此内容升级到最新版本即可正常显示灰色感叹号 icon + 「去升级」按钮
触发条件
场景处理方式
本地文件被删除打开笔记时检测文件是否存在，不存在则切换为空状态
云端文件被删除请求文件信息返回 404 时切换为空状态
网页链接失效后台定期检测（或用户点击时检测），连续 3 次失败后切换为空状态
录音文件被删除同本地文件逻辑，打开笔记时检测
卡片类型无法识别笔记版本低于卡片所需最低版本时，展示未知类型占位
交互规则
操作行为
点击Toast 提示「原文件已被删除，无法打开」，不跳转
长按弹出菜单：重新选择文件 / 移除此卡片
编辑模式下卡片右上角显示 × 按钮，可直接移除
批量操作支持「清理所有失效卡片」（设置页入口）
点击未知类型卡片点击卡片任意区域或「去升级」按钮，跳转应用商店更新页
